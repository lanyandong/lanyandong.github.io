Talk is cheap. Show me the code.

Posted on 2019-11-07 | Edited on 2019-11-08 | In 随笔集 | Comments:

人总是逃不过”真香定理”，之前博客主题从NexT换为Casper，说过不会再换了，这次又把主题换回到NexT，真香啊。这次更换主题最主要的原因还是回归简洁，重新开始。使用最原始的github page，也不想再去弄啥域名映射，反而增加了访问延迟。

Talk is cheap. Show me the code是 Linux 的创始人 Linus Torvalds提到的，我觉得也十分有道理，反观现在各种开源社区的Markdown项目，内容杂乱，有些甚至存在一些错误，就这样传播开来，这样的信息不对称对于初学者无疑是不友好的。我一直认为学习过程中一直存在维度打击，怎么说呢，就是我们对于一个未知的东西是无法甄别其是正确还是错误的，能够规避这个问题的方法也只有以往的知识积累和经验，还有就是书本和官方说明。对于他人的经验和总结，就需要甄别自己受不受用了。

我觉得编程语言只是编程思想的承载者和软件项目开发的工具，熟悉语言就是熟悉工具，而编程更需要的是一种思维训练，很多人都偏向于去学习工具，这无可厚非，毕竟工欲善其事必先利其器。但熟悉了语言用法和特性也不一定写得出代码，了解原理未必能够轻易实现，两者互通有无。思考方式和实现方式需要进行关联，那就是动手实践，我一直认为计算机是一门实践出真知的学科，Talk is cheap. Show me the code. 平时多实践既能够熟悉语言特性也能够了解编程思想，岂不美哉。

我一直苦恼于经常忘记学过的了解过的东西，仔细分析可能有几个原因，一是没有理解其本质，二是没有复习进行反馈，三是没有实践和应用。太急于求成了，输入了大量的信息但没有进行处理和整合，没有积累更没有产出，属于一种空耗，而这正是需要去克服和解决的问题啊。
